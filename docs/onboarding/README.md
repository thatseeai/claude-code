## Claude Code란?

Claude Code는 Anthropic의 공식 AI 코딩 어시스턴트 CLI입니다. 코드 작성, 리뷰, 디버깅, 문서화 등 소프트웨어 개발의 모든 단계를 지원합니다.

### 🎯 주요 특징

- **대화형 인터페이스**: 자연어로 코딩 작업 요청
- **플러그인 시스템**: 확장 가능한 명령어와 에이전트
- **보안 우선**: 내장된 보안 체크와 Hook 시스템
- **Git 통합**: 커밋, 푸시, PR 생성 자동화
- **병렬 처리**: 여러 에이전트가 동시에 작업

### 💡 Claude Code로 할 수 있는 것

```
✓ 새로운 기능 개발 (전체 워크플로우 자동화)
✓ 코드 리뷰 및 버그 발견
✓ 레거시 코드 이해 및 문서화
✓ Git 작업 자동화 (커밋, 푸시, PR)
✓ 리팩토링 및 코드 품질 개선
✓ 테스트 작성 및 실행
```

## 5분 빠른 시작

### 1단계: 설치 확인

Claude Code가 이미 설치되어 있는지 확인:

```bash
claude --version
```

설치되어 있지 않다면:

```bash
npm install -g @anthropic-ai/claude-code
```

### 2단계: 프로젝트로 이동

```bash
cd /path/to/your/project
```

### 3단계: Claude Code 시작

```bash
claude
```

처음 실행하면 이런 화면이 보입니다:

```
Claude Code v1.0.0
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📁 Working Directory: /path/to/your/project
🌿 Git Branch: main
⚡ Model: claude-sonnet-4-5

Available Commands:
  /dedupe           - Find duplicate GitHub issues
  /oncall-triage    - Triage GitHub issues
  /commit-push-pr   - Commit, push, and open a PR

Type your request or /help for assistance.
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

>
```

### 4단계: 첫 번째 질문

간단한 질문으로 시작해보세요:

```
> 이 프로젝트의 구조를 설명해줘
```

또는

```
> README 파일에서 설치 방법을 알려줘
```

### 5단계: 종료

작업이 끝나면:

```
> exit
```

또는 `Ctrl+C` / `Cmd+C`

## 핵심 개념 이해하기

### 🔧 도구 (Tools)

Claude Code는 다양한 도구를 사용하여 작업을 수행합니다:

| 도구 | 용도 | 예시 |
|------|------|------|
| **Read** | 파일 읽기 | 코드 분석, 버그 찾기 |
| **Write** | 파일 생성 | 새 파일 작성 |
| **Edit** | 파일 수정 | 기존 코드 수정 |
| **Bash** | 명령어 실행 | git, npm, 테스트 실행 |
| **Glob** | 파일 검색 | 패턴으로 파일 찾기 |
| **Grep** | 코드 검색 | 특정 함수/변수 찾기 |
| **Task** | 에이전트 실행 | 복잡한 작업 위임 |

**사용자는 도구를 직접 호출하지 않습니다.** Claude가 필요에 따라 자동으로 선택합니다.

### 🤖 에이전트 (Agents)

에이전트는 특정 작업에 특화된 자율적인 AI입니다:

```
code-explorer    → 코드베이스 탐색 및 분석
code-architect   → 아키텍처 설계
code-reviewer    → 코드 리뷰 및 품질 검사
```

에이전트들은 **병렬로 실행**되어 작업 속도를 높입니다.

### 🪝 훅 (Hooks)

훅은 특정 이벤트에 자동으로 실행되는 스크립트입니다:

**SessionStart Hook**
- 세션 시작 시 실행
- 예: 학습 모드 활성화

**PreToolUse Hook**
- 도구 실행 전 실행
- 예: 보안 취약점 체크

**PostToolUse Hook**
- 도구 실행 후 실행
- 예: 자동 포맷팅

### 📂 플러그인 (Plugins)

플러그인은 명령어, 에이전트, 훅을 묶은 패키지입니다:

```
plugins/
├── feature-dev/           # 기능 개발 워크플로우
├── security-guidance/     # 보안 가이드
├── commit-commands/       # Git 명령어
└── code-review/           # 코드 리뷰
```

### 📋 슬래시 명령어 (Slash Commands)

`/`로 시작하는 특수 명령어:

```bash
/feature-dev "OAuth 추가"    # 새 기능 개발
/commit-push-pr              # 커밋 + 푸시 + PR
/code-review                 # 코드 리뷰
```

## 일상적인 워크플로우

### 📝 시나리오 1: 새로운 기능 개발

```
> /feature-dev "사용자 프로필 페이지 추가"
```

**Claude의 7단계 프로세스:**

1. **Discovery**: 요구사항 이해 및 확인
2. **Codebase Exploration**: 관련 코드 탐색 (3개 에이전트 병렬)
3. **Clarifying Questions**: 불분명한 사항 질문
4. **Architecture Design**: 3가지 접근법 제시
5. **Implementation**: 선택한 접근법으로 구현
6. **Quality Review**: 3개 리뷰어가 품질 검사
7. **Summary**: 작업 요약 및 다음 단계 안내

**예상 소요 시간**: 1-2시간 (자동화된 프로세스)

### 🔍 시나리오 2: 코드 이해하기

```
> 이 프로젝트에서 인증이 어떻게 작동하는지 알려줘
```

**Claude의 작업:**

1. 3개 Code Explorer 에이전트 병렬 실행
   - Agent 1: 인증 로직 찾기
   - Agent 2: 데이터베이스 스키마
   - Agent 3: API 엔드포인트
2. 10-20개 핵심 파일 식별
3. 코드 흐름 분석
4. 아키텍처 다이어그램 생성
5. 상세한 설명 제공

**예상 소요 시간**: 30초 - 1분

### 🐛 시나리오 3: 버그 수정

```
> 로그인할 때 세션이 저장되지 않는 버그를 찾아서 수정해줘
```

**Claude의 작업:**

1. 관련 코드 탐색
2. 버그 원인 분석
3. 수정 방법 제안
4. 승인 후 코드 수정
5. 테스트 실행
6. 결과 보고

### 📤 시나리오 4: Git 작업

```
> /commit-push-pr
```

**Claude의 작업:**

1. `git status`로 변경사항 확인
2. `git diff`로 변경 내용 분석
3. 커밋 메시지 작성 (프로젝트 스타일 학습)
4. 파일 스테이징 및 커밋
5. 원격 저장소에 푸시
6. GitHub PR 생성 (상세한 설명 포함)

**예상 소요 시간**: 30초 - 1분

### 📖 시나리오 5: 문서 작성

```
> API 엔드포인트에 대한 문서를 작성해줘
```

**Claude의 작업:**

1. API 엔드포인트 파일 찾기
2. 각 엔드포인트 분석
3. 마크다운 문서 생성
   - 엔드포인트 목록
   - 요청/응답 예시
   - 인증 요구사항
   - 에러 코드

### 일반적인 요청 패턴

**코드 작성:**
```
> UserService에 updateProfile 메서드를 추가해줘
> React 컴포넌트로 프로필 카드를 만들어줘
```

**코드 리뷰:**
```
> 최근 변경사항을 리뷰해줘
> auth.service.ts에서 보안 이슈를 찾아줘
```

**리팩토링:**
```
> UserController의 중복 코드를 제거해줘
> 이 함수를 더 읽기 쉽게 리팩토링해줘
```

**테스트:**
```
> AuthService에 대한 단위 테스트를 작성해줘
> 테스트를 실행하고 실패하는 것을 고쳐줘
```

## 모범 사례

### ✅ 해야 할 것

#### 1. 명확하고 구체적인 요청

**좋은 예:**
```
> UserService의 updateProfile 메서드에 이메일 검증 로직을 추가해줘.
  이메일 형식이 올바른지 확인하고, 중복 이메일이 있는지 체크해야 해.
```

**나쁜 예:**
```
> 뭔가 좀 고쳐줘
```

#### 2. 단계별로 진행

복잡한 작업은 여러 단계로 나누세요:

```
1단계: 사용자 모델을 먼저 수정해줘
2단계: (완료 후) 이제 API 엔드포인트를 추가해줘
3단계: (완료 후) 테스트를 작성해줘
```

#### 3. 변경사항 확인

Claude가 파일을 수정하기 전에 제안을 검토하세요:

```
Claude: "auth.service.ts를 다음과 같이 수정하겠습니다..."
당신: "좋아요, 진행해주세요"
```

#### 4. 정기적으로 커밋

큰 작업은 중간에 커밋하세요:

```
> 여기까지 작업을 커밋해줘
```

#### 5. /feature-dev 활용

새로운 기능 개발 시 `/feature-dev` 명령어를 사용하면:
- 체계적인 7단계 프로세스
- 자동 코드 리뷰
- 품질 보증

### ❌ 피해야 할 것

#### 1. 너무 모호한 요청

```
❌ "프로젝트를 개선해줘"
✓ "UserController의 응답 시간을 개선해줘"
```

#### 2. 한 번에 너무 많은 작업

```
❌ "인증, 프로필, 설정, 알림 기능을 모두 추가해줘"
✓ "먼저 인증 기능부터 추가해줘"
```

#### 3. 보안 경고 무시

Security Hook이 경고하면 주의 깊게 검토하세요:

```
⚠️  보안 경고가 발생했습니다.
→ 무시하지 말고 내용을 확인하세요!
```

#### 4. Git 작업 없이 코드 수정

변경사항은 항상 Git으로 추적하세요:

```
❌ 수정만 하고 커밋 안 함
✓ /commit-push-pr로 자동 커밋
```

## 문제 해결

### 🔧 일반적인 문제

#### 문제: Claude가 파일을 찾지 못함

**증상:**
```
파일 src/components/UserProfile.tsx를 찾을 수 없습니다.
```

**해결:**
```
1. 파일 경로가 정확한지 확인
2. 프로젝트 루트에서 Claude를 실행했는지 확인
3. 파일이 .gitignore에 포함되어 있지 않은지 확인
```

#### 문제: 보안 경고 반복

**증상:**
```
계속해서 같은 보안 경고가 나타남
```

**해결:**
```
보안 경고는 세션당 한 번만 표시됩니다.
새 세션에서는 다시 표시될 수 있습니다.
이것은 의도된 동작입니다.
```

#### 문제: Git 푸시 실패

**증상:**
```
git push 실패: 권한 거부
```

**해결:**
```
1. SSH 키 또는 토큰 설정 확인
2. 원격 저장소 URL 확인: git remote -v
3. 브랜치 추적 설정: git push -u origin <branch>
```

#### 문제: 에이전트가 너무 오래 걸림

**증상:**
```
Code Explorer 에이전트가 5분 이상 실행 중
```

**해결:**
```
1. 검색 범위를 좁히세요:
   "src/auth/ 디렉토리에서만 찾아줘"

2. Thoroughness 레벨을 낮추세요:
   "quick 모드로 빠르게 찾아줘"
```
